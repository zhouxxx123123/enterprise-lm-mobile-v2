## Findings

P0 - 练习和考试作答框架需要按旧移动端原型重构。
旧原型中练习与考试共用连续对话式作答骨架：顶部摄像头/表情识别，练习显示进度，考试显示倒计时；中间是 AI/用户对话流；底部是按住说话；录音中显示说话中动效；结束时弹确认。当前陪练已部分接近，但考试仍是普通题卡表单，和旧原型不一致。

P0 - 考试缺开考前验证/规则层。
旧原型有 `recognition.vue` 人脸验证。当前从考试首页直接进入作答页，缺人脸验证、设备检测、考试规则提示、开始验证状态。

P1 - 陪练缺详情页和角色确认页。
页面清单要求 `陪练详情页` 和 `角色确认页`。当前从首页直接进入作答页，缺场景、AI角色、追问强度、预计时长、训练目标的确认。

P1 - 工作台还偏入口面板，不像员工当前待办工作台。
当前有待办和模块入口，但缺继续进行、最近反馈、报告通知、待复习建议的真实列表。第一屏应优先解决“我现在要做什么”和“我能继续什么”。

P1 - AI对话页的 `+` 菜单不应默认展开。
默认展开遮挡内容并让页面显得复杂。更合理是默认收起，点击 `+` 后浮层展开；常用功能靠近 `+`。

P1 - 考试报告过轻。
报告页只显示状态和两条建议，缺默认报告应有的总分/通过、题目明细、掌握点、待加强、改进建议。可配置不展示分数，但默认应展示完整结构。

P2 - 个人中心抽屉缺个人学习资产结构。
当前只有个人资料、个人上传、建议采纳记录。应补我的学习资产、我的资料、收藏资料、生成内容、自测记录、问答历史、下载管理、消息中心、设置。

P2 - 企业LM首页有轻微层级问题。
底部 AI 入口和学习空间列表抢注意力；继续学习标题右侧 `全部` 与顶部筛选 `全部` 容易混淆。

P2 - 学习空间详情页仍过多展示来源管理。
详情页应围绕学习空间内容和学习进度，不应把资料来源列表做得像管理页。资料来源可以保留，但应弱化，内容列表/课程章节/最近问答更重要。

P2 - 陪练作答页的主动作不够明确。
现在有话术参考、发送、按住说话，但缺“提交当前轮 / 试听 / 重录 / 下一轮”的明确节奏。屏幕阅读器和自动化也会遇到“按住说话”文案重复。

P3 - 部分命名仍像配置说明。
如“考试任务 · 后台可配置”“配置说明”适合内部说明，不适合员工端原型演示。

## Suggested Fix Order

1. 重构考试作答页，按旧原型对话式框架处理演讲题，并加开考前验证/规则页。
2. 补陪练详情页、角色确认页，再优化陪练作答页的提交当前轮节奏。
3. 补工作台继续进行和最近反馈。
4. 收起 AI 对话页默认 `+` 菜单，调整入口层级。
5. 补个人中心学习资产结构。
6. 再做视觉密度和文案统一。
